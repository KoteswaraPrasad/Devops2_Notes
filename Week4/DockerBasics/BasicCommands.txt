options :-

-it -> runs the container in interactive mode, allowing you to interact with the container through the command line
-d -> runs the container in the background
-p -> port number mapping


Running a container with a specific name :
    docker run --name my-container httpd echo "hello world"

Environment Variables 
Need of Environment Variables

    Without -e :
        Configuration must be hard-coded in the image
        Need to rebuild image for every change
    
    With -e :


Setting Environment :
    
    docker run -e MY_VAR=value httpd env
    this command sets an Environment Variable "MY_VAR" to the value "value" and runs a new container from the httpd image

Multiple Environment Variables

Passing Environment Variable from Host System

docker run -d -p 8080:80 --name my_web httpd

8080 port for our host machine
80 port inside container


docker run -d -p 3307:3306 --name mysql-test -e MYSQL_ROOT_PASSWORD=root123 mysql

--name -> naming the container 
-e -> setting Environment Variable

docker exec -it container_id bash/env -> Access the docker container and run commands inside the container in bash

docker rm container_id -> to remove stopped container
docker rm -f container_id -> to remove running containers
docker rmi image_id -> to remove image with no containers
docker rmi -f image_id -> removing image with running containers forcefully
